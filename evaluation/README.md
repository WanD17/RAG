# RAG Evaluation

Automated evaluation của backend RAG. Deterministic metrics + optional LLM judge cho faithfulness.

## File layout

```
evaluation/
├── golden_set.json               # 60 samples (50 in-scope + 10 OOS) — single-turn
├── multiturn_golden_set.json     # 9 conversations, 20 turns — multi-turn
├── upload_docs.py                # [1-time] Register + upload 13 PDFs
├── run_eval.py                   # Single-turn eval runner
├── run_multiturn_eval.py         # Multi-turn eval runner
├── llm_judge.py                  # LLM faithfulness judge (dùng bởi run_eval.py)
├── update_benchmark.py           # Append single-turn result → benchmark.md
├── update_multiturn_benchmark.py # Append multi-turn result → multiturn_benchmark.md
├── compare.py                    # Diff 2 single-turn runs side-by-side
├── benchmark.md                  # Single-turn history log (auto-updated)
├── multiturn_benchmark.md        # Multi-turn history log (auto-updated)
├── README.md                     # This file
├── .config.json                  # JWT + doc IDs (auto-generated by upload_docs.py)
├── .last_run_checkpoint.json     # Resume state (auto-generated)
└── results/
    ├── <timestamp>-<tag>.json              # Single-turn results
    └── <timestamp>-multiturn-<tag>.json    # Multi-turn results
```

## Single-turn metrics

| Group | Metric | Target |
|-------|--------|:------:|
| Retrieval | `doc_hit@5` — GT doc trong top-5 | > 70% |
| Retrieval | `MRR` — rank của GT doc | > 0.5 |
| Answer | `cosine_sim` — sim với reference answer | > 0.6 |
| Answer | `keyword_recall` — required keywords trong answer | > 0.5 |
| Answer | `citation_coverage` — answer có cite source | > 80% |
| Answer | `faithfulness` — LLM judge, optional | > 0.8 |
| Retrieval | `context_precision` — LLM judge, optional | > 0.7 |
| Behavior | `OOS_refusal_accuracy` — OOS refused đúng | > 70% |
| Behavior | `false_refusal` — in-scope refused nhầm | < 10% |
| Latency | `p50 / p95 ms` | p95 < 10s |

**Faithfulness** đo hallucination: answer có chứa claim không có trong retrieved context không.
Khác `cosine_sim` — cosine_sim đo similarity với reference answer, không phát hiện được hallucination.

**Context Precision** đo noise trong retrieval: trong top-k chunks retrieve được, bao nhiêu % thực sự cần thiết để trả lời câu hỏi. Thấp → retriever đang kéo về nhiều chunks thừa → tăng nhiễu cho LLM.

## Multi-turn metrics

| Metric | Mô tả | Target |
|--------|--------|:------:|
| `t1_hit@5` | doc_hit@5 turn 1 (standard) — baseline retrieval | > 70% |
| `coref_kw` | keyword_recall coreference turns — LLM resolve pronoun đúng không | > 60% |
| `ellip_hit@5` | doc_hit@5 elliptical turns — retrieval hoạt động trên câu rút gọn | > 50% |
| `no_leak` | % topic_shift turns không bị topic leakage | > 90% |
| `oos_ref` | % oos_followup turns refused đúng | > 80% |
| `conv_ok` | % conversations có conversation_id nhất quán | 100% |
| `p50/p95 ms` | Latency tính trên tất cả turns | p95 < 10s |

**Test types trong `multiturn_golden_set.json`:**

| test_type | Mô tả | N turns |
|-----------|--------|:---:|
| `standard` | First turn — baseline | 9 |
| `coreference` | Pronoun ("its", "this rate") → retrieval cần resolve | 3 |
| `elliptical` | Câu rút gọn ("And on holidays?") | 3 |
| `continuation` | Deepen cùng topic | 4 |
| `topic_shift` | Đổi topic hoàn toàn, check không bị leak | 1 |
| `oos_followup` | Follow-up OOS sau in-scope turn | 1 |



## Single-turn workflow

### Step 1 — Upload 13 PDFs (1 lần duy nhất)

```bash
python evaluation/upload_docs.py
```

Script tự động register/login, upload PDFs, chờ embedding hoàn thành, lưu JWT vào `.config.json`.

### Step 2 — Smoke test

```bash
python evaluation/run_eval.py --limit 5 --tag smoke
```

### Step 3 — Full eval

```bash
# Deterministic metrics only (nhanh)
python evaluation/run_eval.py --tag baseline

# Với faithfulness judge (llama3.2:3b, chậm hơn ~20%)
python evaluation/run_eval.py --tag baseline --llm-judge
```

**Thời gian** (CPU only, 60 samples):

| LLM | Không judge | Có judge (`llama3.2:3b`) |
|-----|:-----------:|:------------------------:|
| qwen2.5:3b | 30-60 phút | +10-15 phút |
| qwen3:8b | 3-6 giờ | +30-60 phút |

**Interrupt-safe** — Ctrl+C bất cứ lúc nào, resume:
```bash
python evaluation/run_eval.py --resume --tag baseline
```

### Step 4 — Update benchmark

```bash
python evaluation/update_benchmark.py --note "baseline, no reranker"
```

Auto-detect file JSON mới nhất. Idempotent — chạy lại không duplicate.

### Step 5 — Implement improvement → compare

```bash
docker compose restart backend
python evaluation/run_eval.py --tag after-reranker
python evaluation/update_benchmark.py --note "added bge-reranker-base"
python evaluation/compare.py results/20260423-baseline.json results/20260427-after-reranker.json
```

`compare.py` flag `✗ WORSE !!` cho regression > threshold (3% pct, 0.03 scalar, 15% latency).

## LLM Judge — Faithfulness Scoring

### Cách hoạt động

`llm_judge.py` gọi Ollama với prompt yêu cầu chấm điểm 0.0-1.0:
- **1.0** — mọi claim trong answer đều có trong retrieved context
- **0.5** — một số claim được suy diễn ngoài context
- **0.0** — answer chứa thông tin không có trong context (hallucination)

Judge dùng `num_predict=16` (chỉ cần output 1 số) → nhanh hơn nhiều so với generate answer.

### Tại sao phải dùng model khác

Default judge: **`llama3.2:3b`** (Meta) — khác family với backend `qwen3:8b` (Alibaba).

Nếu dùng cùng model, model sẽ tự cho điểm cao hơn thực tế (**self-evaluation bias** ~0.1-0.2).

Override nếu cần:
```bash
python evaluation/run_eval.py --llm-judge --judge-model mistral:7b --tag "with-judge"
```

## Interpret results

### Single-turn

| Symptom | Fix priority |
|---------|:---:|
| `OOS_refusal < 40%` | 🔴 System prompt grounding |
| `citation_coverage < 60%` | 🔴 System prompt citation rule |
| `doc_hit@5 < 60%` | 🟡 Reranker / BM25 hybrid / embed model |
| `cosine_sim < 0.5` nhưng `doc_hit@5 > 70%` | 🟡 Prompt engineering |
| `faithfulness < 0.7` | 🟡 Hallucination — tăng grounding trong system prompt |
| `context_precision < 0.5` | 🟡 Retrieval noisy — tăng score threshold hoặc giảm top_k |
| `false_refusal > 15%` | 🟡 Relax refusal criteria |
| `p95 > 15s` | 🟡 Giảm num_predict hoặc switch model nhỏ |

### Multi-turn

| Symptom | Fix priority |
|---------|:---:|
| `ellip_hit@5` thấp hơn `t1_hit@5` nhiều | 🔴 Implement query rewriter |
| `coref_kw` thấp | 🔴 Query rewriter cần resolve pronoun |
| `no_leak < 90%` | 🟡 Conversation history nhiễu topic shift |
| `conv_ok < 100%` | 🔴 Bug trong conversation manager |
| `oos_ref < 70%` | 🟡 System prompt refusal rule |

## Edit golden sets

### Single-turn (`golden_set.json`)

```json
{
  "id": "q_061",
  "question": "Your question here",
  "reference_answer": "Expected answer with key facts",
  "category": "in_scope",
  "source_doc": "04_labor_code_2019.pdf",
  "source_section": "Article X",
  "required_keywords": ["key", "numbers"],
  "must_refuse": false,
  "difficulty": "easy"
}
```

### Multi-turn (`multiturn_golden_set.json`)

```json
{
  "turn": 2,
  "question": "And for those aged 14 to 16?",
  "reference_answer": "Expected answer...",
  "source_doc": "03_criminal_code_2015.pdf",
  "required_keywords": ["very serious"],
  "forbidden_keywords": [],
  "must_refuse": false,
  "test_type": "elliptical",
  "coref_entity": "optional — what pronoun refers to"
}
```

- `forbidden_keywords` — dùng cho `topic_shift`: nếu answer chứa keyword này = topic leak
- `coref_entity` — chỉ để đọc hiểu, không dùng trong scoring

**Khi nào cần cập nhật:**
- Đổi chunking/embedding/LLM → **không** cần đổi golden
- Thêm PDFs mới → thêm samples liên quan
- Thêm conversation logic mới → thêm conversation vào multiturn set

## Troubleshooting

**`ConnectionError: backend not reachable`**
→ `docker compose ps` + `curl http://localhost:8000/health`

**`token expired / 401`**
→ Xóa `.config.json` và chạy lại `upload_docs.py`

**`Ollama timeout`**
→ `docker compose logs ollama`. Giảm `num_predict` trong `backend/src/rag/generator.py` xuống 512 hoặc switch model nhỏ.

**`[judge error] ...`**
→ `llama3.2:3b` chưa được pull. Chạy: `docker compose exec ollama ollama pull llama3.2:3b`

**`sentence-transformers download slow`**
→ Pre-cache: `python -c "from sentence_transformers import SentenceTransformer; SentenceTransformer('all-MiniLM-L6-v2')"`

**`false_refusal: 100%`**
→ Backend trả lời rỗng. Query thủ công qua UI hoặc curl để debug.
